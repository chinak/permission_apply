模块按职责分为三层：
- 状态层 `store/WarehouseContext.tsx`：通过 React Context + useState 集中管理移仓表单全局状态，定义 `WarehouseExchange`/`WarehouseDirection`/`ContractType`/`PositionRow` 等类型，并提供 `reset`、`toggleAccountPermission`、`hasPermissionForAccount` 等复合 setter；所有页面组件通过 `useWarehouseContext()` 订阅。
- 视图层 `pages/`：`WarehouseList` 展示客户侧申请列表并路由到详情；`WarehouseApply` 承载复杂多步表单（交易所多选、方向分支、合约明细动态行、附件上传），使用本地 `errors` 记录校验结果并通过 `data-error` 属性定位滚动；`WarehouseDetail`/`WarehouseAudit` 分别面向客户只读展示与审核员操作，后者复用同一组件仅以 `isModal` 模式在 `WarehouseAuditList` 的 Dialog 中渲染。
- 辅助层 `components/WarehouseConfigPanel.tsx`：悬浮面板暴露预设场景一键填充，直接调用 Context setter 驱动表单，便于开发/测试快速构造数据。
依赖方向单向：页面 → Context → UI 组件，不反向引用；审批流程采用 Mock 数组模拟营业部→运营中心会签（风控/交割/结算三节点，任一 rejected 即一票否决）→总部经办/复核的状态机。
