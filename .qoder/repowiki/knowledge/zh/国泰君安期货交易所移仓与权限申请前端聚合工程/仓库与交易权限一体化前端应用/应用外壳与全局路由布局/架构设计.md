- 入口 `App.tsx` 仅挂载 `react-router` 的 `RouterProvider`，由 `routes.tsx` 通过 `createBrowserRouter` 声明式定义路由树，所有业务页面作为 Layout 的子路由。
- `layout.tsx` 是根布局组件：外层包裹 `AppProvider`（机构业务通用状态）与 `WarehouseProvider`（移仓业务专用状态），内部实现左侧可折叠导航栏（按「机构业务办理」/「自然人业务办理」分组）、顶部面包屑（基于 `GROUP_BREADCRUMB_MAP` 与 `NAV_GROUPS` 推导）、用户信息区以及 `<Outlet />` 内容区；右侧在首页/补充材料页旁挂 `ConfigPanel`。
- 导航配置采用数据驱动：`NAV_GROUPS` + `NavGroup/NavSubGroup/NavItem` 类型描述菜单层级，`getPageLabel` / `getActiveGroup` 根据当前 pathname 动态计算高亮与面包屑，新增页面只需在 `routes.tsx` 注册并补全映射即可。
- 全局状态分层：`store/AppContext.tsx` 暴露 `useAppContext` Hook，管理适当性评估相关状态（风险等级、资金等级、已选品种等）；移仓业务状态由同目录 `WarehouseContext` 独立维护，二者互不耦合。
- 共享 UI 层：`components/ProcessRecord.tsx` 以时间轴形式展示审批流程记录，`utils/mockData.ts` 提供失败原因等模拟数据源，供配置面板与表单校验复用。
