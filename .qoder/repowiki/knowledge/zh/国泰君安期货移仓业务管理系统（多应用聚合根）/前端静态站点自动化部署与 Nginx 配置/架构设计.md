模块由一个可执行入口 `deploy.sh` 与两份 Nginx 模板组成，无代码依赖，纯声明式配置。
- 入口流程：检查 root 权限 → 校验 `./dist` 存在 → 将当前 `DEPLOY_DIR` 整体 mv 到 `.bak`（先备份后替换）→ `cp -r ./dist/*` 覆盖新文件 → 自动识别 nginx/www-data 用户设置 chown/755 → 用 `sed` 把模板中的 `your-domain.com` 替换为 `SERVER_NAME` 并写入 `$NGINX_CONF_DIR/management-platform.conf` → `nginx -t` 测试失败则自动从 `.bak` 恢复旧目录并退出，成功则通过 `systemctl reload nginx` 或 `nginx -s reload` 热重载。
- 配置层：`nginx.conf` 是主站 server 块模板（含 HTTPS 注释占位、安全头、gzip、/assets 长期缓存、index.html 强制 no-cache、SPA try_files 回退）；`nginx-server.conf` 是 `default_server` 兜底块，用于捕获未匹配域名的请求。
- 产物边界：`dist/` 仅作为只读输入被脚本 rsync/cp 至 `/var/www/management-platform`，脚本不修改其内容。
- 依赖方向：脚本单向依赖系统 Nginx 二进制与 root 能力，Nginx 模板反向引用脚本约定的 `DEPLOY_DIR` 路径，二者通过 `your-domain.com` 占位符 + `sed` 注入建立松耦合。
