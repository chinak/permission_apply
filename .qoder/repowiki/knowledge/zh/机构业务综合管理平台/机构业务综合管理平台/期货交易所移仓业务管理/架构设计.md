- **状态管理层**：以 `WarehouseContext.tsx` 为核心，通过 React Context API 集中管理移仓表单的全局状态（如交易所选择、方向、合约明细、实控账号等），实现页面间数据共享与重置。
- **视图层**：采用页面组件化设计，`WarehouseList`/`AuditList` 负责列表展示与路由跳转，`WarehouseApply` 处理复杂的多步表单逻辑与动态校验，`WarehouseDetail`/`Audit` 分别面向客户与审核员展示只读信息与审批流。
- **交互增强层**：`WarehouseConfigPanel` 作为开发/测试辅助组件，提供一键填充场景数据的快捷入口；`LicenseConfirmDialog` 在提交前介入营业执照合规性确认。
- **数据流向**：表单页通过 Context 驱动 UI 更新，提交时触发校验并弹出确认框；审批页通过 Mock 数据模拟会签节点（一票否决制）的状态流转。
