# 注销交易编码/权限 交互需求文档

> 本文档描述机构业务综合管理平台「注销交易编码/权限」模块的交互需求，聚焦业务流程与交互逻辑，不含具体视觉与 UI 细节。内容基于当前原型系统的实际实现。

---

## 一、功能概述

注销交易编码/权限模块面向内部员工（营业部、运营中心），用于处理客户提交的交易编码或交易权限的注销申请。员工在流水列表中查看待处理申请，进入处理界面对客户申请内容进行核对与调整，并在复核环节完成向 CAP、CTP 系统的注销同步，最终完成审批。

### 1.1 核心业务规则

| 规则 | 说明 |
|------|------|
| 编码与权限的对象区分 | 「注销编码」注销的是交易所交易编码；「注销权限」注销的是具体品种权限（如商品期权、特定品种等） |
| 编码与权限的包含关系 | 注销编码必然导致该编码名下相关权限同步注销；注销权限不一定伴随编码注销 |
| 有权限必有编码 | 客户能勾选某交易所下的具体品种权限，说明其必然持有该交易所编码 |
| 员工操作边界 | 员工只能在客户已提交的申请项基础上取消勾选，不可新增申请项 |

### 1.2 办理状态

| 办理状态 | 含义 |
|----------|------|
| 办理中-经办 | 已到达经办环节，待经办处理 |
| 办理中-复核 | 已到达复核环节，待复核处理（同步操作在此环节执行） |
| 办理失败-经办 | 经办环节办理失败 |
| 办理失败-复核 | 复核环节办理失败 |
| 办理成功 | 全流程办结 |

## 二、流水列表页

| 需求点 | 需求描述 |
|--------|----------|
| 页面结构 | 页面自上而下分为三个区域：顶部筛选条件区、中部数据列表区、底部分页区。 |
| 筛选条件 | 提供流水号、资金账号、姓名（文本输入，模糊匹配）、渠道、营业部、营业部组织架构分组（下拉选择，默认「全部」）、申请日期（日期选择）等筛选项。 |
| 列表字段 | 展示流水号、资金账号、姓名、渠道、营业部、营业部组织架构分组、申请日期、处理日期、最近 CAP 时间、办理状态、经办人、复核人、同步记录等信息。 |
| 状态标识 | 办理状态与同步记录以带色标签区分，便于快速识别处理进度与同步结果。 |
| 行操作 | 每行提供「处理」（打开处理界面）、「备注」（查看/编辑备注）、「下载」（下载该申请全部附件）三个操作入口。 |
| 备注交互 | 点击备注弹出编辑弹窗，可填写并保存备注内容；已有备注的行需在入口做区分提示。 |
| 分页 | 底部显示总记录数与分页控件。 |

## 三、处理审批界面

处理界面以模态框形式打开，自上而下由多个业务模块顺序排列，底部为固定操作按钮区。各模块交互需求如下。

### 3.1 基本资料信息

| 需求点 | 需求描述 |
|--------|----------|
| 概览信息 | 展示资金账号、姓名、营业部、营业部组织架构分组、申请日期、办理状态等概览字段。 |
| 客户资料 | 展示客户身份、联系方式、适当性、休眠状态等基础资料字段；异常或缺失字段需做强提示。 |
| 更新 CAP | 提供「更新 CAP」入口，用于从 CAP 系统拉取并刷新客户最新资料。 |

### 3.2 申请内容

| 需求点 | 需求描述 |
|--------|----------|
| 层级结构 | 按「交易所编码（父级）→ 具体权限（子级）」两级结构展示客户申请注销的内容。 |
| 操作边界 | 员工只能取消客户已勾选的注销项，不能新增；取消后可恢复。 |
| 编码与权限联动 | 勾选注销某交易所编码时，其名下持有的权限随之标记为注销且不可单独调整；取消编码注销后，权限可恢复单独调整。 |
| 子项标识 | 客户无权限的子项标注「（无权限）」且不可操作；有权限但未勾选注销的子项不额外标识。 |
| 中金所特例 | 中国金融期货交易所以「编码及权限」一体化呈现，其编码与权限一并注销，无独立子项时以占位说明维持结构完整。 |

### 3.3 同步操作（CAP / CTP）

| 需求点 | 需求描述 |
|--------|----------|
| 展示条件 | 同步操作区仅在「办理中-复核」状态展示，其他状态不出现。 |
| 操作独立性 | 「提交 CAP：注销编码」与「提交 CTP：注销权限」为两个独立操作，各自统计已选数量、可单独提交。 |
| 已选统计 | 分别实时统计已勾选的待注销交易所编码数量、待注销权限项数量；无可提交项时对应按钮不可用。 |
| 提交确认 | 点击任一提交按钮先弹出确认弹窗，说明本次涉及的编码/权限数量；CAP 确认中需提示「注销编码后名下相关权限将随之注销」。 |
| 提交反馈 | 确认提交后，对应操作旁显示「已提交」标记，并同步写入操作日志（见 3.7）。 |
| 一键批量 | 每个同步操作支持一次点击批量提交对应范围内的全部项，无需逐条操作。 |

### 3.4 交易所编码开通情况

| 需求点 | 需求描述 |
|--------|----------|
| 模块用途 | 展示客户当前已开通的交易所编码情况，供员工核对注销范围。 |
| 展示字段 | 按交易所展示开户日期、休眠状态等信息。 |

### 3.5 附件资料

| 需求点 | 需求描述 |
|--------|----------|
| 附件范围 | 展示客户提交的附件（身份证正反面等）。 |
| 预览 | 支持点击附件直接预览查看。 |
| 下载 | 支持下载附件。 |

### 3.6 全量权限表

| 需求点 | 需求描述 |
|--------|----------|
| 模块用途 | 展示客户当前在各交易所的全量权限持有情况，作为注销操作的对照参考。 |
| 数据加载 | 进入界面时加载展示。 |

### 3.7 操作日志

| 需求点 | 需求描述 |
|--------|----------|
| 模块用途 | 记录各项同步操作的执行情况，供员工在提交同步后核对结果。 |
| 日志项 | 至少包含「同步 CAP-待注销编码」「同步 CTP-待注销权限」「同步德索-材料归档」等操作项。 |
| 记录字段 | 每条记录展示操作内容、操作状态、操作人、操作时间。 |
| CAP/CTP 联动 | 复核环节提交 CAP 或 CTP 后，对应日志项状态由「未同步」变为「同步成功」，并回填操作人与操作时间；两项互不影响。 |
| 德索联动 | 「同步德索-材料归档」在「办理成功」状态下显示为「同步成功」并回填操作人与时间，其余状态显示「未同步」。 |

### 3.8 审批流程

| 需求点 | 需求描述 |
|--------|----------|
| 模块用途 | 以时间轴形式展示该申请的全生命周期流程节点。 |
| 动态渲染 | 根据当前办理状态动态呈现流程：已完成节点、进行中节点、失败节点分别以不同状态标识；办理失败节点需展示失败原因。 |

### 3.9 操作按钮

| 需求点 | 需求描述 |
|--------|----------|
| 位置 | 界面底部固定操作区。 |
| 办理失败 | 点击后弹出原因选择弹窗，选定原因后提交，流水标记为办理失败。 |
| 审批通过 | 点击后弹出确认弹窗，确认后进入下一环节。 |

## 四、状态联动规则汇总

不同办理状态下各交互元素的显隐/状态如下：

| 办理状态 | 同步操作区（CAP/CTP） | CAP/CTP 日志项 | 德索同步日志 |
|----------|----------------------|----------------|--------------|
| 办理中-经办 | 不显示 | 未同步 | 未同步 |
| 办理中-复核 | 显示，可提交 | 未同步 → 提交后同步成功 | 未同步 |
| 办理失败-经办 | 不显示 | 未同步 | 未同步 |
| 办理失败-复核 | 不显示 | 未同步 | 未同步 |
| 办理成功 | 不显示 | 同步成功 | 同步成功 |
