# 交易所移仓业务 产品需求文档

> 本文档专门描述本原型系统中「移仓业务」模块的产品功能与交互逻辑。所有内容基于当前原型系统的实际实现。

---

## 一、功能概述

移仓业务模块是机构业务综合管理平台中的一个子系统，为客户提供期货持仓跨期货公司或同一实控组内账户间转移的在线申请功能，同时为内部员工提供多级审批管理能力。

### 1.1 页面结构总览

| 路由 | 页面 | 面向角色 | 功能说明 |
|------|------|----------|----------|
| `/warehouse-home` | 移仓业务首页 | 客户 | 功能介绍、快速发起入口、申请进度概览 |
| `/warehouse-apply` | 移仓申请表单 | 客户 | 填写并提交移仓申请 |
| `/warehouse-list` | 我发起的申请 | 客户 | 查看历史申请记录及状态 |
| `/warehouse-detail` | 申请详情 | 客户 | 查看单条申请的完整信息及流程记录 |
| `/warehouse-audit-list` | 移仓业务审批流水 | 内部员工 | 审批工作台，管理与审批所有申请 |
| `/warehouse-audit` | 总部审核详情 | 内部员工 | 查看申请详情并执行审批操作 |

### 1.2 支持的交易所

| 交易所 | 高亮色 | 支持的移仓场景 |
|--------|--------|----------------|
| 大连商品交易所（DCE） | 蓝色标签 | 跨期货公司移入/移出；同一实控组内账户间移仓 |
| 郑州商品交易所（CZCE） | 蓝色标签 | 跨期货公司移入/移出 |
| 上海期货交易所（SHFE） | 蓝色标签 | 跨期货公司移入/移出；同一实控组内账户间移仓 |

### 1.3 移仓方向

| 方向 | 含义 | 方向标签色 |
|------|------|-----------|
| 移入我司 | 持仓从其他期货公司转入国泰君安 | 绿色 |
| 移出我司 | 持仓从国泰君安转出至其他期货公司 | 橙色 |
| 实控组移仓 | 同一实际控制关系组内账户之间的持仓转移（仅单独选择大商所或上期所时可用） | 蓝色 |

## 三、移仓申请表单

这是本模块最核心的页面，承载了客户填写并提交移仓申请的全部交互。表单由多个区域组成，表单数据通过全局状态在页面切换时保持。

### 3.1 页面整体结构

表单区域从上到下依次为：

1. **选择移仓的交易所和方向** — 基本信息区
2. **合约明细** — 多行表格编辑区
3. **补充信息与附件** — 选填区
4. **业务办理须知** — 折叠说明区
5. **确认声明** — 勾选确认区
6. **底部提交按钮** — 固定在页面底部
7. **右下角悬浮面板** — 移仓场景面板（开发辅助）

### 3.2 区域一：选择移仓的交易所和方向

#### 3.2.1 交易所选择（多选下拉）

**初始态**：显示灰色占位文字「请选择交易所」。

**展开态**：
- 弹出下拉面板，宽度与触发器对齐
- 顶部为「全选」选项，带复选框
- 下方列出三个交易所选项：大连商品交易所、郑州商品交易所、上海期货交易所，每个带复选框
- 点击选项可勾选/取消

**已选态**：
- 触发器内以蓝色标签展示已选交易所，每个标签右侧带 × 按钮
- 点击标签上的 × 可单独取消该交易所
- 触发器右侧始终显示向下箭头图标

**交互规则**：

| 编号 | 用户操作 | 系统响应 |
|------|----------|----------|
| A-01 | 点击触发器 | 展开/收起下拉面板 |
| A-02 | 勾选/取消某个交易所 | 更新已选列表，标签即时增删 |
| A-03 | 点击标签上的 × | 取消该交易所的选择 |
| A-04 | 点击「全选」 | 若当前未全选，则选中全部三个交易所；若已全选，则取消全部 |
| A-05 | 选中/取消交易所 | 触发合约明细行自动增删（见 3.3.2） |
| A-06 | 选中/取消交易所 | 影响实控组移仓方向的可用性（见 3.2.2） |

#### 3.2.2 移仓方向选择（单选下拉）

**选项列表**：
- 「移入我司（持仓从其他期货公司转入国泰君安）」
- 「移出我司（持仓从国泰君安转出至其他期货公司）」
- 「大商所我司实控组移仓（实际控制关系账户之间移仓）」— **仅在单独选择大商所时显示**
- 「上期所我司实控组移仓（实际控制关系账户之间移仓）」— **仅在单独选择上期所时显示**

**实控组移仓选项的显示规则**：
- 仅当已选交易所数量为 1 且该交易所为 DCE 或 SHFE 时，才出现实控组移仓选项
- 选项文案中的交易所名称随选择动态变化（选择大商所显示"大商所"，选上期所显示"上期所"）
- 如果已选择了实控组移仓方向，但随后交易所选择不满足条件（如多选了交易所），则自动清除方向选择

#### 3.2.3 对手方信息（随方向动态显示）

选择方向后，下方动态出现相应的对手方信息输入区：

**当方向为「移入我司」时**：展示移出方信息
- 移出期货公司会员号（文本输入，必填）
- 移出期货公司名称（文本输入，必填）
- 每个已选交易所对应：移出客户交易编码（文本输入，必填）
- 每个已选交易所对应：移出客户全称（文本输入，必填）

**当方向为「移出我司」时**：展示移入方信息
- 移入期货公司会员号（文本输入，必填）
- 移入期货公司名称（文本输入，必填）
- 每个已选交易所对应：移入客户交易编码（文本输入，必填）
- 移入客户全称（文本输入，必填）
- **申请理由**（多行文本输入，必填，占位文字：「请说明移仓原因」）

**当方向为「实控组移仓」时**：展示实控组账户信息

| 字段 | 输入方式 | 交互行为 |
|------|----------|----------|
| 实控组-移出账号 | 文本输入 + 失焦查询 | 输入后失焦，系统查询该账号信息 |
| 实控组-移出客户全称 | 文本输入（条件只读） | 查询成功时自动填充并变为只读（灰色背景）；无权限时手动填写 |
| 实控组-移入账号 | 文本输入 + 失焦查询 | 同上 |
| 实控组-移入客户全称 | 文本输入（条件只读） | 同上 |

**实控组账号查询逻辑**：

输入账号后失焦，系统根据输入内容返回三种结果：

| 结果 | 提示信息 | 客户全称字段行为 |
|------|----------|-----------------|
| 查询成功 | 无错误提示 | 自动填充客户全称，字段变为只读（灰色背景 + 灰色文字） |
| 账号不存在 | 红色提示：「账号不存在，请检查后重新输入」 | 清空客户全称 |
| 无权限 | 橙色提示：「无权限查看该账号信息」 | 不自动填充，占位文字变为「无权限查看」，字段仍可手动填写 |

**查询重置**：修改账号内容后，查询状态和客户全称自动清空，需重新失焦触发查询。

#### 3.2.4 交易所特定字段

**大商所「是否按量移仓」**（单选下拉）：
- 显示条件：已选交易所包含 DCE，且方向非「移出我司」、非「实控组移仓」
- 选项：「是 / 否」
- 必填

**移出我司「申请理由」**：
- 显示条件：方向为「移出我司」
- 多行文本域，必填

### 3.3 区域二：合约明细

#### 3.3.1 顶部操作栏

- 左侧标题：「合约明细」
- 右侧按钮：「+ 添加一行」，蓝色边框按钮，点击新增一条空白合约明细行

#### 3.3.2 合约明细行自动生成规则

| 规则 | 说明 |
|------|------|
| 首次选择交易所 | 为每个选中的交易所自动生成一条合约明细行，该行的「交易所」字段自动选中对应交易所 |
| 新增选择交易所 | 自动追加该交易所对应的一行 |
| 取消选择交易所 | 自动移除该交易所对应的所有行（手动添加的行不受影响） |
| 所有交易所取消 | 清空所有有交易所归属的行（手动添加的空交易所行保留） |

#### 3.3.3 行内字段

每行包含以下字段，横向排列，可水平滚动：

| 字段 | 控件类型 | 宽度 | 特殊行为 |
|------|----------|------|----------|
| 交易所 | 下拉选择（select） | 120px | 选项：请选择 / 大连商品交易所 / 郑州商品交易所 / 上海期货交易所 |
| 品种名称 | 文本输入 | flex-1 | 必填 |
| 合约代码 | 文本输入 | flex-1 | 必填 |
| 持仓方向 | 下拉选择 | 80px | 选项：买 / 卖 / 全部 |
| 持仓类型 | 下拉选择 | 80px | 选项：投机 / 套保；**选择郑商所时禁用**，显示灰色"—" |
| 手数 | 数字输入 | 80px | 必填，需 > 0；**列标题动态变化**（见 3.3.4） |
| 转移资金（元）| 数字输入（选填）| 120px | 可留空 |
| 操作 | 删除按钮 | — | 红色垃圾桶图标，点击删除该行 |

#### 3.3.4 手数列标题动态规则

手数列的标题根据该行对应的交易所和全局设置动态变化：

| 条件 | 列标题显示为 |
|------|-------------|
| 该行交易所为郑商所（CZCE）| 「预估移仓手数」 |
| 该行交易所为上期所（SHFE）| 「预估移仓手数」 |
| 该行交易所为大商所（DCE）且「是否按量移仓」= 否 | 「预估移仓手数」 |
| 其他情况 | 「移仓手数」 |

**表格列头规则**：当所有行的手数标签一致时，表头显示该统一标签；不一致时显示「移仓手数」。

#### 3.3.5 郑商所特殊规则

选择郑商所后，合约明细区域下方显示橙色提示文字：
> 「郑商所仅支持该合约全部持仓转移，不支持部分转移和区分投机/套保类型」

同时，对应行的「持仓类型」下拉框变为灰色禁用态，显示"—"，不可选择。

#### 3.3.6 空状态

当合约明细列表为空时（无任何行），显示虚线边框区域，居中文字：
> 「暂无合约明细，请点击右上角「添加一行」按钮」

### 3.4 区域三：补充信息与附件（选填）

**补充说明**：
- 多行文本域
- 占位文字：「如有特殊说明，如：实控关系组内转移等，请在此处说明」

**附件上传**：
- 虚线边框的拖拽/点击上传区域
- 图标 + 提示文字：「点击或将文件拖拽至此处上传」
- 副文字：「支持 JPG, PNG, PDF, DOC 格式，单个文件不超过 10MB」
- 点击触发系统文件选择器，支持多选
- 已上传文件以三列网格展示，每个文件卡片包含：文件图标（PDF 红色 / 其他蓝色）、文件名（超长截断）、文件大小
- 每个文件卡片右侧有 × 删除按钮

### 3.5 区域四：业务办理须知（可折叠）

默认折叠状态。点击标题栏展开/收起，箭头图标旋转 180° 动画。

展开后显示 6 条须知，序号列表形式：
1. 移仓前一日结算后可用资金不能为负，账户需处于活跃状态且交易编码正常
2. 移仓转入的品种，您需在我司已具备相应的交易权限
3. 移仓当日结算后若出现可用资金为负，我司可能采取提高保证金、限制出金、冻结资金、强行平仓等措施
4. 移仓后因计算规则差异，逐笔和逐日结算单中的数据可能存在不同
5. 郑商所同一属性及方向的合约持仓，仅支持全部持仓转移，不支持部分转移和区分投机/套保类型
6. 交割、期权行权履约等导致持仓变化的业务，可能使移仓失败或结果与申请内容有出入

### 3.6 区域五：确认声明

- 复选框 + 声明文字
- 声明内容：「本人/本单位确认，以上申请内容真实有效。本人/本单位已知晓并同意贵司有权根据市场行情及账户情况拒绝办理移仓业务，由此带来的一切后果由本人/本单位承担。」
- 未勾选时提交按钮置灰不可用

### 3.7 表单校验

#### 3.7.1 校验触发方式

点击「提交申请」按钮时执行全量校验。

#### 3.7.2 校验规则

| 校验项 | 触发条件 | 错误提示 |
|--------|----------|----------|
| 至少选择一个交易所 | 未选择 | 「请至少选择一个移仓交易所」 |
| 必须选择移仓方向 | 未选择 | 「请选择移仓方向」 |
| 必须填写移出/移入期货公司会员号 | 相应方向下为空 | 「请输入移出/移入期货公司会员号」 |
| 必须填写移出/移入期货公司名称 | 相应方向下为空 | 「请输入移出/移入期货公司名称」 |
| 必须填写各交易所客户交易编码 | 相应字段为空 | 「请输入 XX 交易所移出/移入客户交易编码」 |
| 必须填写各交易所客户全称 | 方向为移入且字段为空 | 「请输入 XX 交易所移出客户全称」 |
| 必须填写移入客户全称 | 方向为移出且字段为空 | 「请输入移入客户全称」 |
| 移出我司需填写申请理由 | 方向为移出且为空 | 「请填写申请理由」 |
| 实控组移出/移入账号必填 | 方向为实控组且为空 | 「请输入实控组-移出/移入账号」 |
| 实控组移出/移入客户全称必填 | 方向为实控组且为空 | 「请输入实控组-移出/移入客户全称」 |
| 大商所是否按量移仓必填 | 方向非移出、非实控组且含 DCE 时为空 | 「请选择大商所是否按量移仓」 |
| 至少一条合约明细 | 合约明细为空 | 「请至少添加一条需移仓的合约」 |
| 合约明细品种名称必填 | 某行的品种名称为空 | 「请输入品种名称」 |
| 合约明细合约代码必填 | 某行的合约代码为空 | 「请输入合约代码」 |
| 合约明细手数必填且 > 0 | 某行的手数 ≤ 0 或为空 | 「手数必须大于 0」 |
| 必须勾选确认声明 | 未勾选 | 「请阅读并勾选确认声明」 |

#### 3.7.3 校验失败响应

- 校验不通过字段的边框变为红色（`border-red-300`）
- 每个字段下方显示红色错误提示文字
- 页面自动滚动到第一个报错字段所在位置（平滑滚动至可视区域中央）
- 提交按钮恢复可点击状态（因校验失败，不会弹出确认框）

### 3.8 提交流程

#### 3.8.1 点击提交

用户点击「提交申请」→ 系统执行校验 → 校验通过后：

1. **弹出营业执照信息确认弹窗**

弹窗内容：
- 顶部蓝色盾牌图标 + 标题「营业执照信息确认」
- 说明文字：「请确认您的营业执照信息是否最新且无误。为保证业务合规性，我们需在最终提交前核对您的资质信息。」
- 左侧：营业执照图片缩略图（固定的示例图片）
- 右侧：关键信息卡片
  - 企业名称（如：超体科技（北京）有限公司）
  - 统一社会信用代码（如：91110000X12345678X）
- 底部黄色警告条：「若上述信息有误，请联系客户经理修改信息后再发起申请」

## 八、状态管理说明

### 8.1 表单状态

移仓申请表单的所有字段状态统一通过 `WarehouseContext` 全局管理：
- 用户在表单中填写的所有数据在页面内切换（如从首页到申请页，从详情页返回列表）时保持
- 仅在手动调用 `reset` 时清空（当前原型未在页面卸载时自动清空）

### 8.2 申请单生命周期

```
[客户填写] → [提交] → [营业执照确认]
    ↓
[营业部审核] ← ← ← [客户重新提交]
    ↓ 通过
[运营中心会签] — 三个岗位并行
    ↓ 全部通过         ↓ 任一驳回 → [已驳回]
[总部经办]                ↓ 驳回
    ↓ 通过
[总部复核]
    ↓ 通过
[已完成]
```

### 8.3 客户可见状态映射

| 内部状态 | 客户列表显示 | 详情页流程记录显示 |
|----------|-------------|-------------------|
| 营业部审核 | 审核中 | 审批中 |

移仓业务首页 (warehouse-home)

    │       │       └── [返回列表] ──→ warehouse-list
    │       ├── [新增移仓申请] ──→ warehouse-apply
